# Product Requirements Document (PRD) for Naver Stock Dashboard

## 1. 소개

### 1.1 제품 개요
Naver Stock Dashboard는 네이버 금융 데이터를 기반으로 한 실시간 주식 정보 제공 웹 애플리케이션입니다. 사용자가 시가총액 상위 50개 주식의 재무 데이터를 조회하고, 개별 주식의 일봉 차트 데이터를 분석할 수 있는 대시보드를 제공합니다.

### 1.2 대상 사용자
- 개인 투자자
- 주식 분석가
- 금융 데이터 연구자

### 1.3 주요 가치 제안
- 실시간 네이버 금융 데이터 접근
- 직관적인 웹 기반 대시보드
- 데이터 분석 및 시각화 기능

## 2. 목표와 목적

### 2.1 비즈니스 목표
- 네이버 금융 데이터를 쉽게 접근할 수 있는 인터페이스 제공
- 주식 데이터 분석을 위한 도구 개발

### 2.2 사용자 목표
- 시가총액 상위 주식의 재무 지표 비교
- 개별 주식의 가격 추이 분석
- 데이터 기반 투자 의사결정 지원

## 3. 기능 요구 사항

### 3.1 핵심 기능

#### 3.1.1 시장 데이터 조회
- **설명**: 코스피 시가총액 상위 50개 주식의 실시간 데이터를 조회
- **입력**: 없음 (실시간 데이터)
- **출력**: JSON 형식의 주식 목록 (순위, 종목명, 코드, 재무 지표들)
- **API 엔드포인트**: `/api/market`

#### 3.1.2 개별 주식 상세 조회
- **설명**: 특정 주식의 일봉 차트 데이터를 조회
- **입력**: 주식 코드 (예: 000660)
- **출력**: JSON 형식의 차트 데이터 (날짜, 시가, 고가, 저가, 종가, 거래량)
- **API 엔드포인트**: `/api/detail/<code>`

#### 3.1.3 데이터 분석
- **설명**: 수집된 데이터를 기반으로 Top 5 분석 수행
- **분석 항목**:
  - 시가총액 Top 5
  - 영업이익 Top 5
  - ROE Top 5
  - PER 저평가 Top 5
  - 외국인비율 Top 5
  - 배당금 Top 5
- **결측치 분석**: N/A 데이터 현황 보고

#### 3.1.4 웹 대시보드
- **설명**: 브라우저 기반 사용자 인터페이스
- **기능**:
  - 종목 리스트 표시 (사이드바)
  - 차트 시각화 (Chart.js 사용)
  - 엑셀 다운로드 기능 (SheetJS 사용)
  - 반응형 디자인 (다크 테마)

### 3.2 보조 기능

#### 3.2.1 데이터 스크래핑
- 네이버 금융 웹사이트에서 데이터 수집
- 세션 관리 및 요청 제한 준수
- 데이터 정제 및 형식 변환

#### 3.2.2 데이터 저장
- JSON 파일로 로컬 저장
- 종목별 데이터 파일 생성

## 4. 비기능 요구 사항

### 4.1 성능
- API 응답 시간: 3초 이내
- 데이터 수집 시간: 요청당 1-2초 대기
- 동시 사용자: 1명 (로컬 개발 환경)

### 4.2 보안
- 네이버 서버 요청 제한 준수
- CORS 설정 (개발 환경)
- 민감한 데이터 노출 방지

### 4.3 사용성
- 직관적인 UI/UX
- 모바일 친화적 디자인
- 오류 처리 및 사용자 피드백

### 4.4 신뢰성
- 네이버 API 변경에 대한 대응
- 예외 처리 및 로깅
- 데이터 유효성 검증

## 5. 사용자 스토리

### 5.1 투자자 페르소나
**이름**: 김투자  
**배경**: 30대 직장인, 주식 초보 투자자  
**목표**: 안전한 투자 종목 찾기

#### 사용자 스토리 1: 시장 개요 파악
김투자는 대시보드를 열어 시가총액 상위 주식을 확인하고, ROE가 높은 종목을 찾는다.

#### 사용자 스토리 2: 종목 상세 분석
김투자는 관심 있는 종목을 선택하여 일봉 차트를 보고, 최근 가격 추이를 분석한다.

#### 사용자 스토리 3: 데이터 내보내기
김투자는 분석한 데이터를 엑셀 파일로 다운로드하여 추가 분석을 수행한다.

## 6. 기술 스펙

### 6.1 기술 스택
- **백엔드**: Python Flask
- **프론트엔드**: HTML, CSS, JavaScript
- **라이브러리**:
  - 데이터 수집: requests, BeautifulSoup
  - 웹 프레임워크: Flask, Flask-CORS
  - 차트: Chart.js
  - 엑셀: SheetJS (xlsx)
- **데이터 형식**: JSON

### 6.2 아키텍처
- RESTful API 디자인
- 클라이언트-서버 모델
- 로컬 파일 기반 데이터 저장

### 6.3 데이터 소스
- 네이버 금융 웹사이트 (https://finance.naver.com)
- 네이버 모바일 API (https://api.stock.naver.com)

### 6.4 배포 환경
- 로컬 개발 서버 (Flask 내장 서버)
- 호스트: 0.0.0.0, 포트: 5000

3. UI/UX 개선
